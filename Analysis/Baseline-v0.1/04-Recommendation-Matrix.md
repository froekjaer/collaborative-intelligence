## Recommendations Not Accepted as Written

### Archive Mission Platform immediately

**Source:** Z.ai.

**Disposition:** Not accepted as written.

The repository is weak in its present state, but it contains potentially useful architecture tests and meta-model ideas. Freezing, correcting and requiring one executable compatibility artefact is more reversible and evidence-producing than archival.

### Delete or merge specified framework documents immediately

**Source:** Z.ai.

**Disposition:** Not accepted as written.

Redundancy and cognitive load are credible findings, but file deletion without a complete authority/content map may destroy distinct context. Map, classify and measure overlap before consolidation.

### Create a minimum viable implementation in every repository immediately

**Source:** Mistral.

**Disposition:** Not accepted as written.

Not every repository needs executable code. Mission Framework and Collaborative Intelligence may legitimately own semantics and programme records. Implementation should be added only where it enforces or tests a declared responsibility.

### Treat all test gaps as conventional code-test gaps

**Source:** Primarily Mistral and Z.ai.

**Disposition:** Modified.

Documentation and semantic repositories require conformance, link, schema, recovery and traceability tests rather than artificial unit tests for prose.

## Recommended Execution Order

1. Freeze the reviewed state through a baseline manifest and coordinated tags.
2. Publish this synthesis and create disposition records.
3. Correct authority boundaries and review structure.
4. Freeze or repair Mission Platform semantics.
5. Implement the minimal conformance spine.
6. Execute the Solar Eclipse vertical slice and capture findings.
7. Reassess conceptual content, Platform scope and documentation consolidation using empirical evidence.
8. Begin brown-field guidance and the next controlled evaluation round.

## Acceptance Rule

No recommendation becomes canonical merely by appearing in this matrix. Each accepted recommendation must become one of:

- an ADR;
- a Framework Finding disposition;
- a tracked backlog item with owner and acceptance evidence;
- a mission decision record;
- a release/baseline change with immutable reference.
